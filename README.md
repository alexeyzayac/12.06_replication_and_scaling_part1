### Задание 1

На лекции рассматривались режимы репликации master-slave, master-master, опишите их различия.

*Ответить в свободной форме.*

#### Решение:

**Master-Slave**
- Один узел — **master** принимает все записи. Другие узлы — **slave** получают данные от master и обычно используются для чтений. Репликация чаще асинхронная, но возможна и синхронная.

*Плюсы:*
* Повышает производительность за счёт распределения операций чтения.
* Позволяет использовать **slave** для резервного копирования и аналитических запросов.
* Простая и понятная архитектура.

*Минусы:* 
* **Master** является единой точкой отказа для операций записи.
* При выходе **master** из строя требуется ручное или полуавтоматическое переключение.
* Возможны задержки репликации, из-за чего данные на **slave** могут быть неактуальны.

**Master-Master**
- Два и более узла принимают записи одновременно. Репликация двунаправленная — каждый мастер реплицирует изменения другим мастерам.

*Плюсы:*
* Высокая доступность: отказ одного **master** не останавливает запись.
* Можно распределять нагрузку на запись между несколькими узлами.
* Подходит для геораспределённых систем с локальной записью.

*Минусы:* 
* Возможны конфликты данных при одновременной записи на разных узлах.
* Требуется сложная логика разрешения конфликтов.
* Более сложная настройка и сопровождение по сравнению с **master-slave**.
* Повышенный риск рассинхронизации при сетевых сбоях.
